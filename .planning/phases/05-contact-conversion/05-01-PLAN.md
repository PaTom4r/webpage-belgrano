**Step 3 — Create `src/lib/rate-limiter.ts`:**
In-memory Map: key = IP string, value = `{ count: number; resetAt: number }`.
Export `checkRateLimit(ip: string): { allowed: boolean }`.
Logic: if entry missing or `Date.now() > resetAt`, reset to count=1, resetAt=Date.now()+3600000, return allowed=true.
If count >= 5, return allowed=false.
Otherwise increment count, return allowed=true.
Note in comment: resets on server restart/redeploy — acceptable for an agency contact form.

**Step 4 — Extend `src/lib/schemas/contact.ts`:**
Add honeypot field to schema: `honeypot: z.string().max(0, 'Bot detected').optional()`.

Preserve existing name/email/company/message fields exactly.

**Step 5 — Create `src/app/actions/contact.ts`:**
Mark file with `'use server'` at top.
Import: `Resend` from `'resend'`, `render` from `'@react-email/components'`, `contactSchema` from schemas, `checkRateLimit` from rate-limiter.
Import email templates (created in Task 2): `TeamNotificationEmail`, `LeadConfirmationEmail`.

Export async function `submitContact(formData: ContactFormData): Promise<{ success: boolean; error?: string }>`.

Implementation order inside function:
1. Honeypot check: if `formData.honeypot && formData.honeypot.length > 0`, return `{ success: false, error: 'invalid' }` silently.
2. Zod server-side validation: `const parsed = contactSchema.safeParse(formData)`. If `!parsed.success`, return `{ success: false, error: 'Datos inválidos. Por favor revisa el formulario.' }`.
3. IP rate limit: get IP from `headers()` (import from `'next/headers'`). Use `x-forwarded-for` header or fall back to `'unknown'`. Call `checkRateLimit(ip)`. If `!allowed`, return `{ success: false, error: 'Demasiados intentos. Intenta de nuevo más tarde.' }`.
4. Resend send: `const resend = new Resend(process.env.RESEND_API_KEY)`. Send both emails in parallel with `Promise.all`:
   - Team notification to `process.env.TEAM_EMAIL`
   - Lead confirmation to `parsed.data.email`
   Use `await render(...)` to get HTML from react-email templates.
5. If Resend throws, catch and return `{ success: false, error: 'Error al enviar. Por favor escríbenos a contacto@belgrano.cl' }`.
6. On success, return `{ success: true }`.
